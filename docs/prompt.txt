***Task Overview***
Given a choropleth map image, extract the required information, evaluate the map’s quality and return the response as JSON only. No additional text is allowed outside the JSON output.
This map’s intended audience is {{audience}}, and its intended purpose is {{purpose}}.
Uploaded data distribution summary, if provided: {{distribution}}.
Context-use requirement: If the intended audience is not "unknown", explicitly mention that audience in the "map_quality" field. If the intended purpose is not "unknown", explicitly mention that purpose in the "map_quality" field. Recommendations must be phrased for the stated audience and purpose, not as generic map-design advice.
***Instructions***
Carefully read the guidelines below to determine what information must be extracted from the map.
Analyze the map to extract each required data item. Track the qualities (good/bad/neutral/meta) associated with each item. All items in the “metadata” subsection of the Required Data Items section must be classed as “meta”. Every item in the “inferred_interpretation” and “normative_evaluation” sections has quality rationale defined — classify each as good, bad, or neutral strictly according to its rationale. Use “neutral” only when the rationale explicitly allows it (e.g., when evidence is genuinely ambiguous). Do not default to neutral.
Evaluate critically: the purpose of this system is to help users improve their maps, so identifying weaknesses is valuable. Do not give “good” unless the criterion is clearly and fully met — if something is merely adequate, passable, or only partially meets the standard, classify it as “bad”. When uncertain between “good” and “bad”, lean toward “bad” so the user receives actionable feedback. Always provide concrete recommendations for every “bad” item in the fixes field.
IMPORTANT: Evaluate the map strictly from the perspective of the intended audience, not from your perspective as an AI. Do not use your own knowledge or inference to fill in gaps. If something is not explicitly shown, labeled, or written on the map, the viewer cannot know it. For example, if a colour on the map has no legend entry, it is unexplained to the viewer even if you can guess what it means. Judge only what the map itself communicates.
For the provided description of the map’s data distribution, assess whether the binning (classification) method shown in the map is suitable for that distribution, and include this assessment in the JSON output.
In addition to the JSON-formatted output, append the following textual summaries inside the JSON fields:
A factual explanation of the map contents (up to 100 words). Highlight only meaningful key facts or concrete findings with double asterisks, e.g. **unemployment rate by county** or **missing source date**. Do not highlight single filler words, transition words, generic phrases, or function words.
A 300-word evaluation explaining whether the map is good or bad, based on extracted information and the count of good/bad items. This field must include one sentence that begins with "For the intended audience, ..." when audience is provided, and one sentence that relates the judgment to the stated purpose when purpose is provided. Highlight only specific quality judgments or evidence-based findings with double asterisks, e.g. **legend is incomplete**, **classification method is unclear**, or **colors are distinguishable**. Do not highlight isolated words such as "while", "because", "map", "data", or "good".
Include a 150-word recommendations section summarizing how the map can be improved based on any criteria marked “bad”. Recommendations must be tailored to the intended audience and purpose when they are provided. Highlight only specific recommended actions with double asterisks, e.g. **add a source date** or **label the no-data category**. If one or more criteria are bad, provide only actionable recommendation sentences and do not include the word “none”. If no criteria are bad, the entire recommendations field must be exactly “none”.
Confirm that the final JSON is syntactically correct and includes all fields in the structure shown below. Do not output anything outside this JSON.
